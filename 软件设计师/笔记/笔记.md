## 位示图

### 计算位示图大小

**磁盘容量为 1024GB，物理块大小为 8MB，字长为 128位**

#### 1. 确定磁盘物理块数量

- 磁盘容量为 1024GB，物理块大小为 8MB
- 因此，磁盘上的物理块数量为：

$$
\frac{1024GB}{8MB} = \frac{1024 \times 1024MB}{8MB} = 131072\:块
$$

每个物理块占用1bit

#### 2. 确定位示图需要位数

- 每个物理块需要一个位来表示是否被占用，故位示图需要 131072 位

#### 3. 计算位示图占用的字数

- 所以

$$
\frac{131072\:位}{128\:位/字} = 1024\:块
$$

### 磁盘存储的核心时间参数

| 术语         | 定义                                             | 影响因素                               |
| ------------ | ------------------------------------------------ | -------------------------------------- |
| 磁盘转速     | 磁盘盘片每分钟旋转的圈数(RPM)                    | 硬件设计，直接决定旋转速度             |
| 旋转等待时间 | 磁头等待目标扇区旋转到下方的平均时间(约半周时间) | 仅与磁盘转速成反比                     |
| 平均寻道时间 | 磁头移动到目标磁道的平均时间                     | 磁头机械臂速度、调度算法与转速无关     |
| 数据传输速率 | 单位时间内从磁盘读取或写入的数据量               | 转速、位密度、接口带宽等多因素共同作用 |

磁盘总存取时间=寻道时间+旋转等待时间 +数据传输时间



### 输入输出（I/O）控制方式

| I/O方式      | 关键特征                                    | CPU 干预程度     | 适用场景           |
| ------------ | ------------------------------------------- | ---------------- | ------------------ |
| 程序查询方式 | CPU 主动轮询外设状态，效率低                | 高（全程参与）   | 简单、低速率设备   |
| 程序中断方式 | 外设主动通知CPU,CPU保存现场后处理数据       | 中等(传输时千预) | 中低速设备，如键盘 |
| DMA方式      | DMA 控制器直接管理数据块传输，CPU仅处理起止 | 低(仅启动和结束) | 高速设备，如磁盘   |

保护现场：在中断发生时，CPU保存当前程序状态(如寄存器值)，以便中断处理后能恢复执行

并行工作：指外设和CPU能同时操作，而非CPU等待外设，从而提升系统吞吐率。



### 事件

| 事件类型        | 来源                   | 典型例子                          | 关键特征                           |
| --------------- | ---------------------- | --------------------------------- | ---------------------------------- |
| 中断(Interrupt) | 处理器外部(如外设)     | DMA传送结束、键盘输入、定时器到期 | 异步，与当前指令无关，可屏蔽       |
| 异常(Exception) | 处理器内部(指令执行中) | 除数为0、缺页异常、非法操作码     | 同步，由当前指令引起，通常不可屏蔽 |



### 寻址方式

| 寻址方式   | 定义                                      | 速度 | 关键特点                       |
| ---------- | ----------------------------------------- | ---- | ------------------------------ |
| 立即寻址   | 操作数直接包含在指令中                    | 最快 | 无需额外访问，但操作数大小受限 |
| 寄存器寻址 | 操作数位于CPU寄存器中，指令指定寄存器编号 | 次快 | 访问速度快，但寄存器数量有限   |
| 直接寻址   | 指令中直接给出操作数的内存地址            | 最慢 | 可访问任意内存位置，但速度慢   |

### 排序

| 排序方法     | 第一趟排序后的关键行为                 | 是否能确定元素最终位置         |
| ------------ | -------------------------------------- | ------------------------------ |
| 直接插入排序 | 将前两个元素比较并排序，构建初始有序段 | 否，元素可能被后续插入操作移动 |
| 冒泡排序     | 通过相邻交换将极值(如最大值)移至一端   | 是，极值位置被固定             |
| 简单选择排序 | 选择最小元素交换到序列起始位置         | 是，起始位置元素确定           |
| 堆排序       | 建堆后交换堆顶元素与末尾元素           | 是，交换后的末尾元素位置固定   |
| 快速排序     | 划分序列，基准元素置于正确位置         | 是，基准元素位置确定           |
| 归并排序     | 将序列分成子序列并两两归并             | 否，元素在递归归并中可能移动   |

排序算法可分为选择类(如简单选择、堆排序)和交换类(如冒泡、快速排序)，它们通常每趟确定一个元素的最终位置;而插入类(如直接插入)和归并类算法则逐步构建全局有序，元素位置需多趟后才能稳定。
